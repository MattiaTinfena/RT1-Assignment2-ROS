scripts.actionClientNode.feedbackHandler(feedback)[source]

Callback to handle feedback from the action server.

scripts.actionClientNode.getCoordinate(prompt)[source]

Prompts the user to input a valid coordinate.

This function continually asks for user input until a valid number is entered. It will also exit the program if the user enters ‘e’.

Parameters:

prompt (str) – The prompt message to display to the user.

Returns:

The valid coordinate input by the user or “exit” if the user chooses to exit.

Return type:

float or str

scripts.actionClientNode.publishStatus(odomMsg)[source]

Publishes the robot’s current position and velocity.

This function receives an Odometry message, extracts the robot’s position and velocity, and publishes it to the ‘/robot_status’ topic.

Parameters:

odomMsg (Odometry) – The Odometry message containing the robot’s position and velocity.

scripts.actionClientNode.sendGoal(client, xTarget, yTarget)[source]

Sends a goal to the action server.

This function sends the target coordinates to the action server and initiates goal tracking. It also sets up the feedback callback function.

Parameters:

  • client (actionlib.SimpleActionClient) – The action client instance that communicates with the action server.

  • xTarget (float) – The X coordinate of the target goal.

  • yTarget (float) – The Y coordinate of the target goal.

scripts.actionClientNode.userInteraction(client)[source]

Handles user inputs for goal management.

This function provides a simple interface for the user to interact with the action client. The user can cancel goals, request feedback, or exit the program.

Parameters:

client (actionlib.SimpleActionClient) – The action client instance.

Returns:

A string indicating the action taken, or “exit” to stop the program.

Return type:

str

scripts.serviceNode.handleTargetRequest(request)[source]

Callback function for the service that handles requests for the last target coordinates.

This function retrieves the target coordinates from the ROS parameter server and returns them as a response to the service request.

Parameters:

request (lastTargetRequest) – The service request. It is not used in this case, but is required by the service signature.

Returns:

The last target coordinates (X and Y).
